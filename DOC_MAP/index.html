    Guiding documents → repository behaviors¶

    This map is the Phase 0 traceability layer. When you change a guiding document, update the corresponding behaviors (rules, templates, or this map) in the same change.

    Guiding document What it governs Concrete behaviors
    VISION_AND_SCOPE.md Mission, boundaries, what belongs in-repo README framing; issue labels; scope checks in AGENTS and Cursor rules
    METHODOLOGY.md Claims vs hypotheses, evidence bars, workflows PR template checklist; finding_review and hypothesis_thread issue templates; prompt templates in prompts/
    ETHICS_REPRODUCIBILITY_AND_DATA.md Data classes, privacy, reproducibility .gitignore; data/README.md; no raw restricted data in git
    QUALITY_BAR.md Explicit quality / anti-sloppiness playbook CI gates; review lanes; def. of done; links METHODOLOGY.md, COLLABORATION_AND_REVIEWS.md, HAPPY_PATH_FIRST_RECORDS.md
    COLLABORATION_AND_REVIEWS.md Contributions, review expectations CONTRIBUTING.md; PR template; branch protection notes in OPERATING_RHYTHM.md
    OPERATING_RHYTHM.md Cadence, versioning, when to sync rules/CI; milestone doc + contributor hub Doc/CI updates; release/tag policy; CHANGELOG, dashboard/index.html, http://localhost:8765/dashboard/ check per .cursor/rules/documentation-and-dashboard.mdc
    REPOSITORY_MANIFEST.md Per-path roles and governing docs Keeping structure auditable; updates when paths change
    ONBOARDING.md First-hour navigation Points to all core policies and work directories
    HAPPY_PATH_FIRST_RECORDS.md Stream A: first unknown + hypothesis YAML → PR CONTRIBUTING.md; schemas/README.md; templates/README.md; CI validates via scripts/validate_schemas.py
    LABELS_AND_MILESTONES.md GitHub triage vocabulary Aligns with COLLABORATION_AND_REVIEWS.md; issue templates
    LICENSING_NOTES.md Exceptions to default MIT Per-file or per-artifact license metadata
    CODE_OF_CONDUCT.md Community behavior CONTRIBUTING.md; enforcement expectations
    SECURITY.md Private vulnerability / sensitive data reports Automation and secrets; not scientific debate
    DATA_PLAN.md Phase A external metadata pilot; envelope schema Must stay consistent with ARCHITECTURE.md ingestion section and LEGAL.md before Phase B jobs land
    examples/README.md Checked-in synthetic ingest JSONL exemplar + regen note Traceable to DATA_PLAN.md stretch item; rows validated in packages/ingest/tests/test_envelope_schema.py
    UAT_INGEST.md Manual acceptance checks for usdr-ingest Implements DATA_PLAN.md QA; automated checks in packages/ingest/tests
    DEV_DASHBOARD.md Meta checklist: active branches, shipped items, blockers Edits .planning/STATE.md; optional sync to Canvas via scripts/sync-dashboard-from-state.py; HTML dashboard + dashboard/README.md for browser use
    GSD_INTEGRATION.md Optional get-shit-done / spec-driven workflows in Cursor Boundaries vs METHODOLOGY.md, DATA_PLAN.md, LEGAL.md; does not replace contributor onboarding

    Supporting files (see manifest)¶

    Configuration and templates (.github/, .cursor/, .markdown-link-check.json, root LICENSE, .gitignore) are indexed in REPOSITORY_MANIFEST.md so every tracked path maps to policy.

    Gaps flagged for maintainer decision¶

    • License interaction: Repository default is MIT (LICENSE). Record per-artifact exceptions in LICENSING_NOTES.md when you introduce them.
    • Human subjects / regulated research: If work involves human subjects, HIPAA, or export-controlled data, add a mandatory compliance appendix and legal review before committing study protocols or data paths.
    • Branch protection: GitHub branch protection is configured in repository settings, not in git. See OPERATING_RHYTHM.md.
