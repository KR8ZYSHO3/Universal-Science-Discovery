# USDR Early Stewards — lightweight charter (v0.1)

**Purpose:** Distribute ownership without bureaucracy. Stewards are the first external cohort that helps USDR stay credible, curated, and welcoming.

**Recruitment:** [Discussion #286 — Early Stewards call](https://github.com/KR8ZYSHO3/Universal-Science-Discovery/discussions/286)

---

## What a steward is

A domain-respected contributor who commits **1–3 hours/month** to:

1. **Triage** open issues and Wave Factory drafts in their domain cluster
2. **Review** 1–2 PRs per month (YAML quality, schema, citation hygiene)
3. **Surface** one high-leverage open unknown or bridge gap per quarter
4. **Optional:** host one 60-min contribution office hour per year

Stewards are **not** maintainers with merge rights by default. They advise, review, and recruit.

---

## What stewards get

- Named credit in preprint acknowledgments and steward lineage (`pioneers/` / governance docs)
- Early invite to the first Unknowns Hackathon (virtual)
- Direct line to founding maintainer for roadmap input
- Visibility on dashboard / release notes when they land substantive contributions

---

## Target first cohort (3–5 people)

| Cluster | Steward responsibilities |
|---------|-------------------------|
| Physics / complex systems | Percolation, criticality, spin-glass bridges; Crosscheck repro review |
| Life sciences | Ecology, neuroscience, aging unknowns; clinical/bench feasibility flags |
| CS / AI for science | Harvesters, schemas, submission portal; metascience tooling |
| Social / climate science | Tipping points, opinion dynamics, cascade unknowns |
| Open science / libraries | Institutional partnerships, citation, onboarding docs |

One steward per cluster is enough for launch. Expand when PR volume justifies it.

---

## Decision rights (keep narrow)

**Stewards recommend.** **Maintainers merge.**

Stewards can:

- Label issues (`launch-sprint`, domain tags)
- Approve or request changes on contributor PRs (via GitHub review)
- Propose steward candidates in their network

Stewards cannot:

- Change schemas or CI without maintainer PR
- Auto-promote Wave Factory batches
- Speak as official USDR legal/funding representatives
